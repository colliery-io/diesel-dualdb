## Objective

Stand up the crate so the bridge work has a place to live: a library crate that compiles with both backend features, with `DualConnection` generated by `#[derive(diesel::MultiConnection)]`. This is the foundation the rest of the spike builds on; it also forces the first real version/feature decisions.

## Acceptance Criteria

## Acceptance Criteria

- [x] crate (`diesel-dualdb`) builds clean (wrote files directly instead of `cargo init`, to avoid `git init` in a non-repo dir).
- [x] `diesel` dependency pinned with `postgres`, `sqlite`, `r2d2`, `uuid` features; `uuid` crate added. Version + intended MSRV noted in status below. **Pin is 2.3.x, not 2.2.x — see status.**
- [x] `#[derive(diesel::MultiConnection)] pub enum DualConnection { Pg(PgConnection), Sqlite(SqliteConnection) }` compiles, generating `MultiBackend`.
