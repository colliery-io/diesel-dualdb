# Uuid portable type: Pg + SQLite ToSql/FromSql + concrete round-trip

## Parent Initiative

[[DDB-I-0001]]

## Objective

Build the Uuid portable type against the two **concrete** backends and prove it round-trips on each, before involving `MultiBackend`. This isolates the per-backend serialization (the well-understood part) from the bridge plumbing (the risky part in [[DDB-T-0003]]). Establishes the marker/newtype conventions the whole M0 type layer will follow.

## Acceptance Criteria

## Acceptance Criteria

- [x] `sql_types::Uuid` marker (`src/sql_types.rs`): derives `SqlType` + `QueryId`, annotated `#[diesel(postgres_type(name = "uuid"))]` and `#[diesel(sqlite_type(name = "Binary"))]`.
- [x] `types::Uuid(pub uuid::Uuid)` newtype (`src/types/uuid.rs`): derives `AsExpression` + `FromSqlRow` with `#[diesel(sql_type = crate::sql_types::Uuid)]`; plus `From` conversions to/from `uuid::Uuid`.
- [x] `ToSql`/`FromSql` for `Pg` — delegate to diesel's native uuid support.
- [x] `ToSql`/`FromSql` for `Sqlite` — 16-byte BLOB encode/decode (not TEXT).
- [x] Round-trip test on a **concrete** `SqliteConnection` (in-memory): insert + read back the full mixed row, assert equality.
- [x] Round-trip test on a **concrete** `PgConnection`, gated on `DUALDB_PG_URL` (skips cleanly if unset; ran for real under `angreal test all`).
- [x] nil and max uuid values both round-trip (plus a v4 case).

## Implementation Notes

### Technical Approach
Reference `../cloacina/src/database/universal_types.rs` for an existing hand-rolled SQLite uuid encoding to crib/validate against. SQLite stores the raw 16 bytes via `Vec<u8>`/`&[u8]` BLOB binding.

### Dependencies
[[DDB-T-0001]] (crate + derive must compile).

### Risk Considerations
Low — this is per-backend `ToSql`/`FromSql`, the well-trodden Diesel path. The marker-type annotation names (`sqlite_type(name = "Binary")`) are the one spot to get exactly right; wrong affinity surfaces as a metadata mismatch in [[DDB-T-0003]].
