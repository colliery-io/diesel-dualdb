# Crate hygiene & feature flags

## Parent Initiative

[[DDB-I-0002]]

## Objective

Turn the spike crate into a properly-configured library: per-type Cargo features over optional deps, the `returning_clauses_for_sqlite_3_35` diesel feature locked in as required, a verified MSRV, and a feature matrix that actually builds. Establishes the gating convention every subsequent type task follows.

## Acceptance Criteria

## Acceptance Criteria

- [x] Crate features `uuid`/`chrono`/`serde_json`, each = `["dep:<x>", "diesel/<x>"]` (also enables diesel's matching integration). `default = ["uuid","chrono","serde_json"]`. `uuid` gates `src/types/uuid.rs` + `sql_types::Uuid` + the bridge.
- [x] `uuid` dep optional; `types::Uuid`, the marker, and the bridge all `#[cfg(feature = "uuid")]` — the retrofit pattern for later types. Bridge gating done via a `#[cfg(feature = "uuid")] mod uuid { … }` in `src/backend.rs` so its imports are scoped too.
- [x] `returning_clauses_for_sqlite_3_35` kept **non-optional** with an explanatory comment; diesel's per-type integrations moved out of the always-on list into our features.
- [x] Feature matrix compiles — `angreal check features` runs `cargo check` for `--no-default-features`, default, each feature alone, and `--all-features`. All green.
- [x] **MSRV set to 1.86** — diesel 2.3.10 declares `rust-version = "1.86.0"`, which binds us (provisional 1.84 was too low). Couldn't bisect below stable (only stable+nightly installed), but diesel's floor is the binding constraint regardless.
- [x] `cargo clippy --all-targets --all-features -D warnings` clean; metadata added (keywords, categories). `repository`/`homepage` left as TODO until the repo URL is settled.

Uuid-dependent test files (`tests/{types,bridge,pool}.rs`) carry `#![cfg(feature = "uuid")]` so `--no-default-features` builds clean; `tests/connectivity.rs` is feature-agnostic.

## Implementation Notes

### Technical Approach
Optional deps via `dep:uuid` etc.; type modules gated with `#[cfg(feature = …)]` in `src/types/mod.rs` and the bridge sections in `src/backend.rs`. For MSRV, pin a toolchain (e.g. `cargo +1.XX check`) downward until it breaks, or read diesel 2.3's documented MSRV and adopt the max of (ours, diesel's). The feature-matrix check is a good first `angreal` task beyond test.

### Dependencies
First M0 task; everything else (CI, type tasks) assumes the feature convention this sets. No blockers.

### Risk Considerations
Low. Main trap: forgetting to gate the bridge impl alongside the type module, which breaks `--no-default-features`. The matrix check catches it.
