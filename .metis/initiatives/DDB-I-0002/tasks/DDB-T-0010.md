# Json<T> portable type (concrete + bridge + tests)

## Parent Initiative

[[DDB-I-0002]]

## Objective

Add `types::Json<T>(T)` (serde round-trip) as a full vertical slice. Pg native `jsonb`, SQLite `TEXT`. The trickiest concrete impl of the three because of Pg's `jsonb` binary wire format. Last type in M0.

## Acceptance Criteria

## Acceptance Criteria

- [x] `sql_types::Json` marker (`jsonb` / `Text`), `#[cfg(feature = "serde_json")]`.
- [x] `types::Json<T>(pub T)`, serde_json-gated, + `From<T>`.
- [x] Concrete impls:
  - `Pg`: writes `0x01` version byte then `serde_json::to_writer`; reads by checking `bytes[0] == 1` and `from_slice(&bytes[1..])` ‚Äî hand-rolled the version byte directly (cleaner than the Value round-trip detour, and keeps `T` generic without a `Value` hop).
  - `SQLite`: `serde_json::to_string` / `from_str` over `TEXT`.
- [x] Bridge: three impls on `MultiBackend`, generic over `T`.
- [x] Round-trip tests (`tests/json.rs`), both backends: null, empty object/array, deep+unicode object, mixed array, **and a typed struct**.
- [x] One-arm ORM test through `DualConnection`, both backends (macro runs all four ways).
- [x] SQLite native JSONB out of scope ‚Äî TEXT only.

**Bound finding:** the **MultiBackend** `ToSql` impl needs `T: Send + Sync + 'static` (the bind collector boxes the value across the backend enum). The concrete `Pg`/`Sqlite` impls need only `T: Serialize + Debug`. `Debug` is required because diesel's `ToSql` has a `Debug` supertrait. Also added a `serde` optional dep (the `serde_json` feature now enables it) + a `serde` dev-dep with `derive` (test-only, for the typed struct).

## Implementation Notes

### Technical Approach
Pg: easiest correct path is to go through diesel's existing `Jsonb`/`serde_json::Value` `To/FromSql` (which already handle the 0x01 version byte) and convert `T` ‚Üî `serde_json::Value` at our layer ‚Äî avoids hand-rolling the wire prefix. SQLite: plain `to_string`/`from_str`. Generic over `T` means the marker is non-generic (`sql_types::Json`) while the newtype carries `T`. Watch the trait bounds on the bridge impls (they'll be generic over `T: Serialize + DeserializeOwned`).

### Dependencies
[[DDB-T-0006]] (`serde_json` feature gate). Bridge pattern from [[DDB-T-0003]]. Independent of Timestamp/Bytes.

### Risk Considerations
Medium ‚Äî the generic `T` makes the bridge impls generic (slightly more involved than the unit types), and Pg `jsonb`'s version byte is a known footgun if hand-rolled (delegating to diesel's `Jsonb` avoids it). Round-trip with a real typed struct, not just `Value`, to prove the generic path.

Last M0 type done. All four portable types (Uuid, Bytes, Timestamp, Json) now ship.

**Result:** `angreal test all` ‚Üí **20 tests green** (Json adds 4). `angreal check all` clean ‚Äî feature matrix incl. `--features serde_json` alone, fmt, clippy.

**Findings:**
- Hand-rolling the Pg `jsonb` `0x01` version byte (write `&[1]` then `to_writer`; read `bytes[0]==1` then `from_slice`) is simpler than routing through `serde_json::Value` and avoids a needless allocation/conversion ‚Äî and keeps `Json<T>` truly generic.
- The MultiBackend `ToSql` bound is heavier than the concrete ones: `T: Send + Sync + 'static` (bind collector boxes across the enum). Documented inline. Concrete impls only need `Serialize + Debug` (Debug because `ToSql: Debug`).
- Same column (`doc -> Json`) round-trips both `serde_json::Value` and a typed struct ‚Äî the generic path works end to end.
