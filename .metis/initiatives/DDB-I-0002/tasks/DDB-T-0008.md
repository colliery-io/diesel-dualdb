# Timestamp portable type (concrete + bridge + tests)

## Parent Initiative

[[DDB-I-0002]]

## Objective

Add `types::Timestamp(DateTime<Utc>)` as a full vertical slice — marker, concrete `Pg`/`Sqlite` impls, the `MultiBackend` bridge, and tests — following the [[DDB-T-0003]] pattern. Postgres uses native `timestamptz`; SQLite uses fixed-format RFC3339 TEXT per [[DDB-A-0001]]. The first type to exercise the **ordering** guarantee, not just round-trip.

## Acceptance Criteria

## Acceptance Criteria

- [x] `sql_types::Timestamp` marker (`timestamptz` / `Text`), `#[cfg(feature = "chrono")]`.
- [x] `types::Timestamp(pub chrono::DateTime<Utc>)` (`src/types/timestamp.rs`), chrono-gated, + `From` conversions.
- [x] Concrete impls: `Pg` delegates to diesel's `Timestamptz` (tz-aware); `SQLite` uses the single `SQLITE_FORMAT` const (`%Y-%m-%dT%H:%M:%S%.6fZ`) for format, `parse_from_rfc3339().with_timezone(&Utc)` for parse.
- [x] Bridge: three one-liner impls on `MultiBackend` (chrono-gated `mod timestamp` in `backend.rs`).
- [x] Round-trip tests (`tests/timestamp.rs`), both backends: subsecond, epoch, pre-1970 (1950), far-future (2999).
- [x] **Ordering test:** insert out of order, `ORDER BY at` returns chronological — passes on SQLite (where stored TEXT is compared directly) and Pg.
- [x] One-arm ORM test through `DualConnection`, both backends (macro runs all four ways).
- [x] Sub-microsecond truncation tested (nanos → micros) and documented in the type's rustdoc.

**Extra finding:** `diesel::table!` syntactically recognizes a column named `Timestamp` as temporal and emits `column ± Interval` operators, requiring the marker to impl `diesel::sql_types::ops::{Add,Sub}` — added (Rhs = `Interval`, Output = `Timestamp`), mirroring native types. Not needed for Uuid/Bytes (non-temporal names).

## Implementation Notes

### Technical Approach
SQLite side: `out.set_value(dt.format("%Y-%m-%dT%H:%M:%S%.6fZ").to_string())`; parse back with the matching `DateTime::parse_from_rfc3339`/`NaiveDateTime` then assert UTC. Keep the format string in one `const`/fn so it can never drift (ADR's load-bearing constraint). Reuse the `tests/types.rs` + `tests/bridge.rs` table patterns from the spike. cloacina's `UniversalTimestamp` (`/Users/dstorey/Desktop/cloacina/crates/cloacina/src/database/universal_types.rs`) is a reference, but note it stored Pg as `Timestamp` not `Timestamptz` — we want tz-aware.

### Dependencies
[[DDB-A-0001]] (format must be locked first) and [[DDB-T-0006]] (the `chrono` feature gate). Bridge pattern from [[DDB-T-0003]].

### Risk Considerations
Medium (highest of the type tasks). The ordering guarantee is the real test; a wrong/variable format passes round-trip but fails ordering. The dedicated ordering test is mandatory. Pg `timestamptz` vs `timestamp` (naive) confusion is a classic bug — assert tz handling explicitly.

## Status Updates

### 2026-06-08 — Complete

Timestamp slice done; ADR ordering guarantee verified on both backends.

**Files:** `src/sql_types.rs` (Timestamp marker + `ops::Add`/`Sub`), `src/types/timestamp.rs`, `src/types/mod.rs`, `src/backend.rs` (chrono-gated bridge), `tests/timestamp.rs`.

**Result:** `angreal test all` → 16 tests green (Timestamp adds 4). `angreal check all` clean — feature matrix incl. `--features chrono` alone, fmt, clippy.

**Findings:**
- **`diesel::table!` + type name `Timestamp`** → emits temporal `± Interval` operators needing `ops::Add`/`Sub` on the marker. Mirrored native impls (Rhs=Interval, Output=Timestamp). A future type whose name collides with a diesel-recognized temporal/numeric name will hit the same; document for M1/macro work.
- SQLite `ToSql` delegates via `out.set_value(String)`; `parse_from_rfc3339` round-trips the canonical format including `Z` and 6-digit subseconds.
- Ordering holds: lexicographic TEXT == chronological, exactly as ADR [[DDB-A-0001]] designed. Truncation: nanos→micros on both backends (Pg `timestamptz` is also µs).

**Handoff:** only [[DDB-T-0010]] (Json) and [[DDB-T-0007]] (CI) remain in M0. Json is the last/trickiest type (generic `T`, Pg jsonb version byte).
