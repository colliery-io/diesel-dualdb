## Implementation Notes

### Technical Approach
Two viable shapes for the Postgres side: (a) GitHub `services: postgres:16` with health-check options and `DUALDB_PG_URL=postgres://…@localhost:5432/…` — the cleanest in Actions, no Docker-in-Docker; or (b) install Docker + run `angreal test all` to reuse the exact local path. (a) is simpler and faster; the per-test skip logic means the same test files work unchanged. Note the libpq/libsqlite3 system-lib needs (spike [[DDB-T-0001]] found system libs locally) — CI runners have them, but pin if needed.

### Dependencies
[[DDB-T-0006]] (feature-matrix check to call). The orchestration ([[DDB-T-0005]]) already exists. Can run in parallel with the type tasks.

### Risk Considerations
Low. Main decision is compose-in-CI vs Actions-service; both work. Avoid flakiness by using the Postgres service health check before tests start.
