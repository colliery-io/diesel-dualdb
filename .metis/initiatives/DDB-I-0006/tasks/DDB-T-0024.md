# Decimal portable type (numeric / TEXT)

## Parent Initiative

[[DDB-I-0006]]

## Objective

A portable `Decimal` type: exact arbitrary-precision numerics on both backends. PG native `numeric`; SQLite canonical decimal string as TEXT. Mirrors the Timestamp slice.

## Acceptance Criteria

- [x] Newtype `types::Decimal(BigDecimal)` — PG arm delegates to diesel's `Numeric`; SQLite arm stores/parses the canonical string. `From`/`Into` `BigDecimal`.

## Implementation Notes

### Technical Approach
Followed the Timestamp template exactly (native PG / TEXT SQLite). `bigdecimal` 0.4 matched to diesel's `numeric` feature (cargo unifies, so `BigDecimal: ToSql<Numeric, Pg>` resolves to one type). SQLite side uses `BigDecimal::to_string()` / `from_str` for exact round-trip.

### Risk Considerations
PartialEq only (not Eq/Hash) — avoids bigdecimal scale-equality subtleties. SQLite-side ordering/arithmetic on the TEXT is not guaranteed (documented); it's a value store. Precision/scale typmods not enforced on SQLite.
