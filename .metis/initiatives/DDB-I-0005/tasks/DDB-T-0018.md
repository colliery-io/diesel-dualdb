# Packaging, dogfood &amp; docs (angreal task, replace inline schemas, run story)

## Parent Initiative

[[DDB-I-0005]]

## Objective

Make the generator usable and trustworthy: a one-command `angreal` task to run it, the crate dogfooding its own output (generated `schema.rs` replacing the inline test `table!`s), the `diesel migration run` story documented, and CI proving the committed output is fresh. This is what closes the schema-gen MVP.

## Acceptance Criteria

## Acceptance Criteria

- [x] **`angreal schema gen`** (+ `angreal schema check`) in `.angreal/task_schema.py` — thin wrapper over the `schema-gen` bin (`cargo run -p schema-gen -- schema/migrations schema/generated`). Decision: bin + angreal wrapper (matches the existing task style).
- [x] **Dogfood:** logical source `schema/migrations/2026-01-01-000000_init/up.sql` (a `widgets` table exercising all four portable types) → committed `schema/generated/` (schema.rs + both migration trees). `tests/generated.rs` `include!`s the generated `schema.rs` **and** `include_str!`s the generated per-backend `up.sql`, then a `#[diesel_dualdb::test]` runs the generated migration + round-trips a row — **both backends green**. (See note on "replace inline".)
- [x] **Run story documented:** README "Schema & migrations" section — logical DDL → `angreal schema gen` → `diesel migration run --migration-dir … per backend` → query the generated `schema.rs`; covers FK→`joinable!`, composite PK, and the escape hatch.
- [x] **CI freshness:** `angreal schema check` (generate to temp + `diff -r` against committed) **and** a `.github/workflows/ci.yml` "Schema freshness" step. Also bumped CI clippy + test to `--workspace`.
- [x] `angreal check all` clean (fmt, clippy `--workspace`, feature matrix). actionlint clean on the workflow.

**Deviation from "replace the inline `table!`s":** kept the per-type test files (`tests/uuid.rs` etc.) — they're individually feature-gated and provide the `--no-default-features`/per-feature matrix coverage that a single generated all-features schema can't. Instead added `tests/generated.rs`, which *uses* the generated `schema.rs` + generated migrations end-to-end (the real proof the AC wanted). Net test count up (now 22 + 2 dogfood + 3 schema-gen).

## Implementation Notes

### Technical Approach
Thin angreal wrapper over the `schema-gen` bin (subprocess), mirroring the existing `task_test.py`/`task_check.py` style. Dogfood: pick which test tables to migrate (uuid_items/bytes_items/ts_items/json_items + the FK fixture from [[DDB-T-0017]]); generate into `tests/generated_schema.rs` (or a `schema/` dir) and `include!` it. The CI freshness check is `generate to a temp dir && diff against committed` → non-zero on drift.

### Dependencies
[[DDB-T-0016]] + [[DDB-T-0017]] (the generator must be capable enough to express the test schemas, incl. the FK fixture). Last task of the MVP.

### Risk Considerations
Low-medium. The dogfood migration is the real proof and may surface generator gaps → loop back to T-0016/T-0017. Keep generated output stable/deterministic so the CI freshness check isn't flaky.
