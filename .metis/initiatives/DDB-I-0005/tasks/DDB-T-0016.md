# Generator core: migrations dir → per-backend trees + schema.rs

## Parent Initiative

[[DDB-I-0005]]

## Objective

Grow the spike ([[DDB-T-0015]]) into a real generator: read a directory of logical-DDL migrations and emit (a) a Postgres migration tree, (b) a SQLite migration tree, and (c) a unified multi-table `schema.rs`. This is the MVP heart — one logical source → everything an adopter needs to build and query their DB on both backends.

## Acceptance Criteria

## Acceptance Criteria

- [x] **Input:** `read_migrations(dir)` reads `<version>_<name>/up.sql` (+ optional `down.sql`), sorted by dir name; multiple `CREATE TABLE`s per file.
- [x] **Per-backend trees:** `migrations-postgres/<dir>/{up,down}.sql` + `migrations-sqlite/<dir>/{up,down}.sql` with native types; `down.sql` derived as reverse-order `DROP TABLE IF EXISTS` when absent. Verified end-to-end via the CLI on a real dir.
- [x] **`schema.rs`:** `@generated` header + one `table!` per table across all migrations, portable markers, grouped `use` (single-item imports rendered without braces for clean compilation when dogfooded).
- [x] **Multi-table / multi-file:** test covers two migrations / two tables in version order.
- [x] **Errors not panics:** unmapped type → `Err("...no mapping for type...")` with table/column context + `migration <name>` prefix; bin exits non-zero. Test asserts it.
- [x] Deterministic output (sorted import names, stable column order).
- [x] Tests green (`cargo test -p schema-gen`, also under `angreal test all`/`sqlite` now that the test task runs `--workspace`).

**Structure:** refactored `schema-gen` into `src/lib.rs` (model + `parse_tables`/`generate`/`render_*`/`read_migrations`/`write_outputs`) + a thin `src/main.rs` CLI (`schema-gen <migrations_dir> <out_dir>`). `.angreal/task_test.py` now runs `cargo test --workspace`.

## Implementation Notes

### Technical Approach
Build on `schema-gen/src/main.rs`. Factor into a lib: `parse_migration_dir() -> Vec<Table>` (+ later ALTERs), `render_pg()/render_sqlite()/render_schema_rs()`, and a thin `main` (`schema-gen <migrations_dir> <out_dir>`). Keep the `map_type` table central. Use `walkdir`/`std::fs` for the dir; sort by version prefix. `down.sql`: reverse-order `DROP TABLE IF EXISTS`. Decide the out-dir layout (sibling `migrations-postgres`/`migrations-sqlite` + `schema.rs` path).

### Dependencies
[[DDB-T-0015]] (spike, done — its `map_type` + renderers are the seed). Feeds [[DDB-T-0017]] (coverage) and [[DDB-T-0018]] (packaging/dogfood).

### Risk Considerations
Medium. `sqlparser` handles parsing; risk is in the breadth of DDL (deferred to [[DDB-T-0017]]) and getting the diesel migration tree layout exactly right so `diesel migration run` accepts it. Keep down.sql generation conservative.

**Decisions:**
- Out layout: sibling `migrations-postgres/` + `migrations-sqlite/` trees + a `schema.rs` file under the out dir.
